2. Diviso Treino/Teste - PRIMEIRA Etapa (Metodologia Correta)

Por que dividir ANTES da preparao?

Data Leakage um dos erros mais graves em ML:

  • Se normalizarmos ANTES de dividir, usamos estatsticas do teste no treino
  • O modelo "conhece" indiretamente os dados de teste
  • Performance inflada e irreal

Metodologia Profissional:

  1. Separar primeiro: Dados "vistos" vs "no vistos"
  2. Preparar s o treino: Calcular estatsticas s dos dados de treino
  3. Aplicar no teste: Usar as mesmas transformaes do treino

Estratificao:

  • Mantm a proporo das classes igual em treino e teste
  • Importante para datasets desbalanceados
  • Garante representatividade em ambos os conjuntos

3. Criao do Pipeline - Automatizao Profissional

O que um Pipeline?

Um Pipeline uma cadeia automatizada que:

  • Agrupa todas as etapas de preparao e treinamento
  • Aplica transformaes na ordem correta
  • Evita vazamento de dados (data leakage)
  • Facilita reproduo e deployment
  • Organiza o cdigo de forma profissional

Componentes do nosso Pipeline:

  1. Tratamento de categricas (se necessrio)
  2. Seleo de atributos (SelectKBest)
  3. Normalizao (StandardScaler)
  4. Treinamento (RandomForest)

Vantagens:

  • Uma linha de cdigo para treinar tudo
  • Consistncia garantida entre treino e teste
  • Cdigo mais limpo e profissional
  • Fcil de modificar e experimentar
In[16]:
# Criando o Pipeline completo de Machine Learning
print("Construindo Pipeline automatizado...")

# Nosso pipeline ter 3 etapas sequenciais:
# 1. Normalizao dos dados
# 2. Seleo das melhores features
# 3. Treinamento do classificador

pipeline_ml = Pipeline([
    # ETAPA 1: Normalizador (coloca tudo na mesma escala)
    ('normalizacao', StandardScaler()),
    
    # ETAPA 2: Seletor de Atributos (escolhe as 20 mais importantes)
    ('selecao_features', SelectKBest(
        score_func=f_classif, 
        k=20
    )),
    
    # ETAPA 3: Classificador (o algoritmo que vai aprender)
    ('classificador', RandomForestClassifier(
        n_estimators=100,
        max_depth=10,
        min_samples_split=5,
        random_state=42
    ))
])

print("Pipeline construdo com sucesso!")
print("\nEtapas do Pipeline:")
for i, (nome, etapa) in enumerate(pipeline_ml.steps, 1):
    print(f"   {i}. {nome}: {type(etapa).__name__}")

4. Treinamento do Pipeline - Uma Linha de Cdigo

A Magia do Pipeline:

Todo o processo que antes levava vrias etapas separadas na Somativa 1, agora acontece com um nico comando: .fit().

Mtodo Anterior (Parte I):

# Mltiplas etapas manuais e sequenciais
X_processado = scaler.fit_transform(X_treino)
X_selecionado = seletor.fit_transform(X_processado, y_treino)
modelo.fit(X_selecionado, y_treino)

Mtodo com Pipeline (Parte II):

# Uma linha para executar tudo na ordem correta
pipeline_ml.fit(X_treino, y_treino)

O que acontece internamente:

Ao chamar .fit(), o pipeline executa a sequncia que definimos:

  1. Aplica fit_transform do StandardScaler nos dados de treino (X_treino).
  2. Aplica fit_transform do SelectKBest nos dados j normalizados.
  3. Aplica fit do RandomForestClassifier nos dados finais (normalizados e selecionados).

Tudo isso de forma automtica, na ordem correta e sem vazar informaes dos dados de teste.

5. Predies com Pipeline - Automao e Consistncia

Consistncia Garantida:

A maior vantagem de usar o pipeline para predies a garantia de que os dados de teste passaro pela exata mesma preparao que os dados de treino.

  1. Normalizao Idntica: Usa a mdia e o desvio padro calculados no treino para normalizar o teste.
  2. Mesmas Features: Aplica a seleo de atributos aprendida no treino para filtrar o teste.
  3. Modelo nico: Usa o RandomForestClassifier j treinado para fazer a predio final.

Sem Data Leakage e Simulao Real:

O pipeline garante que os dados de teste (X_teste) sejam tratados como "dados novos e nunca vistos". As transformaes (StandardScaler, SelectKBest) usam apenas o .transform(), sem reaprender com o teste. Isso simula perfeitamente como o modelo se comportaria em um ambiente de produo.

6. Avaliao do Modelo com Mtricas Profissionais

Para avaliar a performance do nosso modelo, no basta apenas olhar as predies. Precisamos de mtricas quantitativas. Escolhemos duas mtricas principais para este problema de classificao.

Mtricas Escolhidas:

1 Accuracy (Acurcia):

  • O que mede: A porcentagem total de acertos do modelo.
  • Frmula: (Verdadeiros Positivos + Verdadeiros Negativos) / Total de Amostras
  • Por que usar: a mtrica mais intuitiva e fcil de comunicar. Responde pergunta: "Das vezes que o modelo tentou prever, quantas ele acertou?".

2 F1-Score:

  • O que mede: Uma mdia harmnica entre Preciso e Recall. uma mtrica mais robusta que a acurcia, especialmente se as classes forem um pouco desbalanceadas.
  • Preciso (Precision): Dos que o modelo previu como 'T', quantos eram realmente 'T'?
  • Recall (Revocao): De todos os 'T' que realmente aconteceram, quantos o modelo conseguiu encontrar?
  • Por que usar: Oferece uma viso mais balanceada da performance, penalizando o modelo se ele for bom em uma mtrica (ex: preciso) mas ruim em outra (ex: recall).

Estas duas mtricas, juntas, fornecem uma viso completa e confivel do desempenho do nosso classificador.

7. Anlise Crtica dos Resultados

Nesta seo, vamos analisar as mtricas obtidas para avaliar a real performance do nosso modelo e entender suas foras e fraquezas. Conforme a orientao, a avaliao no considera somente o resultado, mas principalmente a forma como chegamos a ele.

Anlise das Mtricas Principais

  • Accuracy (Acurcia): O valor da acurcia nos d a porcentagem geral de acertos. Uma acurcia alta (geralmente acima de 70-75% para este tipo de problema) sugere que o modelo significativamente melhor do que um palpite aleatrio.

  • F1-Score: Esta mtrica combina preciso e recall. Um F1-Score prximo da acurcia indica que o modelo tem um desempenho equilibrado para ambas as classes (CT e T), sem favorecer excessivamente uma em detrimento da outra. Se o F1-Score fosse muito mais baixo que a acurcia, isso seria um sinal de alerta de que o modelo pode estar ignorando uma das classes.

  • AUC (rea Sob a Curva ROC): A AUC mede a capacidade do modelo de distinguir entre uma classe e outra. Um valor prximo de 1.0 indica uma excelente capacidade de separao, enquanto um valor de 0.5 representa um desempenho aleatrio.

Interpretao da Matriz de Confuso

A Matriz de Confuso nos mostra exatamente onde o modelo est acertando e errando:

  • Verdadeiros Positivos (TP) e Verdadeiros Negativos (TN): Representam os acertos do modelo. Queremos que estes nmeros sejam os mais altos possvel.
  • Falsos Positivos (FP) e Falsos Negativos (FN): Representam os erros. Um "Falso Positivo" ocorre quando o modelo prev 'T' e o resultado real era 'CT'. Um "Falso Negativo" o oposto, prevendo 'CT' quando o real era 'T'. Analisar qual desses erros mais comum pode revelar um vis no modelo.

Veredito Preliminar

Com uma acurcia de aproximadamente 74.9% e um F1-Score de 0.74, consideramos que o modelo tem um bom desempenho inicial. Os valores so positivos porque demonstram que o algoritmo conseguiu aprender padres relevantes nos dados para prever o vencedor do round com uma taxa de acerto considervel. A proximidade entre a acurcia e o F1-Score sugere que a performance balanceada entre as equipes CT e T.

  3. O processo iterativo: Este resultado nos d um modelo campeo, mas o ciclo de melhoria poderia continuar, talvez testando uma gama ainda maior de parmetros ou introduzindo novos algoritmos.
